能查天气、能算数学、能记住你的名字。**核心 109 行**（< 150）。

## 它揭穿了什么（下篇文章的核心论点）

所谓 Agent，内核就是一个循环：

```
用户提问
  → 大模型决定：调哪个工具，还是直接回答      （think）
  → 我们执行那个工具                          （act）
  → 把工具结果喂回给模型                       （observe）
  → 重复，直到模型给出最终答案
```

就这么一圈。对应到代码里：

| 概念（LangChain 里的大词） | 这里的真身 |
|---|---|
| LLM | `call_llm()` —— 一次 HTTP POST |
| Tools | 四个普通 Python 函数 + 一个 `TOOLS` 字典 |
| Memory | 一个全局 `dict`（`MEMORY`） |
| Agent / 编排 | `run_agent()` 里的那个 `for` 循环 |
| Prompt | `SYSTEM` 那段字符串（教模型用 JSON 表达意图）|

**LangChain 帮你包的，正是这一圈。** 拆开看，它没那么神——但你会真正地懂它：
框架省的是"把这些接口统一、调好"的功夫；而原理，就上面这张表。

## 给文章的延伸点（写的时候可用）
- "模型怎么知道调哪个工具？" → 不是魔法，是 `SYSTEM` 提示词约定了"输出 JSON"，我们 `extract_json` 解析它。
- "为什么要 `temperature=0`？" → 要它稳定、可解析，不要它发挥。
- "为什么限制循环 8 步？" → 防止模型陷入"反复调工具"的死循环（真实 Agent 也要这个护栏）。
- "安全：`eval` 不危险吗？" → 用正则只放行数字与运算符 + 清空 `__builtins__`，演示了"工具要带护栏"。
- 可作钩子：下下篇可加"多工具协作 / 让它自己写代码并执行"，自然延伸到多 Agent。
